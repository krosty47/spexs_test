# Changelog - Week 1, 02-04-2026, V. 0.6.0

**Release Date**: Week 1, 02-04-2026 at 18:24
**Version**: 0.6.0 (previously 0.5.0)
**Object**: feat: Nodemailer SMTP migration, HTML email templates, user selector, and IN_APP recipient routing

## Changes

### Backend

- **Mailer migration**: Replaced Resend (API-key-based) with Nodemailer (SMTP-based). Users configure their own SMTP settings instead of depending on a third-party API key.
- **Graceful degradation**: When SMTP is not configured, email sending is skipped with a warning log. Notifications default to SSE/in-app only.
- **HTML email templates**: Created table-based, email-client-compatible templates with shared `baseLayout()` wrapper for: event triggered, event resolved, event snoozed, and daily summary emails.
- **Config feature flags**: New `config.getFeatures` tRPC query exposes `{ emailEnabled }` based on `SMTP_HOST` presence, protected by AuthMiddleware.
- **Users endpoint**: New `users.list` tRPC query returns registered users filtered by current user and system user (`notIn: [currentUserId, 'system']`).
- **IN_APP recipient routing**: `EventsService.trigger()` now sends in-app notifications to all IN_APP recipients (by user ID) plus the workflow owner, deduplicated via `Set`.
- **DRY improvement**: Extracted duplicated transport creation logic in `MailerModule` into `createTransportProvider()`.
- **Env validation**: Replaced `RESEND_API_KEY`/`RESEND_FROM` with `SMTP_HOST`, `SMTP_PORT`, `SMTP_SECURE`, `SMTP_USER`, `SMTP_PASS`, `SMTP_FROM`.

### Frontend

- **Email option toggle**: EMAIL recipient channel is conditionally rendered based on `config.getFeatures.emailEnabled`.
- **User selector dropdown**: Replaced free-text destination input with a `<select>` populated by `trpc.users.list`. Shows `Name (email)` for all channels. Value is user ID for IN_APP, email for EMAIL.
- **Channel switch reset**: Changing the channel type resets the destination selection.
- **Root redirect**: Added `app/page.tsx` redirecting `/` to `/workflows`.

### Shared

- **Config schema**: `appConfigOutputSchema` with `emailEnabled` boolean.
- **User schema**: `userListItemSchema` and `userListOutputSchema` for the users endpoint.

### Tests

- 130 backend tests passing
- Updated event trigger tests to verify IN_APP notifications sent to all recipients
- Updated mock data: IN_APP destinations are now user IDs instead of emails
