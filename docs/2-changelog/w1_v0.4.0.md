# Changelog - Week 1, 02-04-2026, V. 0.4.0

**Release Date**: Week 1, 02-04-2026 at 15:18
**Version**: 0.4.0 (previously 0.3.0)
**Object**: feat: notifications system v0.4.0 - in-app notifications, SSE real-time delivery, email notifications on event trigger

## Changes

### Backend - Notifications Module

- New `Notification` Prisma model with `NotificationType` enum (EVENT_TRIGGERED, EVENT_RESOLVED, EVENT_SNOOZED, EVENT_REOPENED)
- `NotificationsService` refactored from single broadcast Subject to user-scoped `Map<userId, Subject[]>` for isolated SSE streams
- Public facade: `NotificationsService.send()` persists notification to DB + pushes SSE event in one call
- CRUD: `findAllForUser()` (paginated, unreadOnly filter), `getUnreadCount()`, `markAsRead()` (with ownership validation), `markAllAsRead()`
- `markAsRead` optimized to single `updateMany` query with FORBIDDEN/NOT_FOUND fallback
- tRPC router (`notifications.*`) with `list`, `unreadCount`, `markAsRead`, `markAllAsRead` procedures
- SSE controller protected by `JwtCookieGuard` (extracted to `features/auth/jwt-cookie.guard.ts`)
- `JwtCookieGuard` validates JWT payload fields (sub, email, role) before trusting them
- `NotificationsModule` imports `AuthModule` (which re-exports `JwtModule`) instead of duplicating JWT config

### Backend - Event Trigger Integration

- `EventsService.trigger()` creates in-app notification for workflow owner via `NotificationsService.send()`
- Email notifications sent to EMAIL channel recipients via `ResendEmailService` (fire-and-forget with `Promise.allSettled`)
- Recipients parsed with `z.array(recipientSchema).safeParse()` for runtime safety on JSON field
- `where` clauses typed with `Prisma.EventWhereInput` / `Prisma.NotificationWhereInput`

### Backend - Tests

- 20 new `NotificationsService` unit tests (subscribe, notify isolation, send persistence, findAll, unreadCount, markAsRead ownership, markAllAsRead)
- 5 new `EventsService` trigger tests (email sending, non-blocking failure, in-app notification, empty recipients, no-EMAIL recipients)
- Test helper `setupTriggerMocks()` extracted to reduce boilerplate
- All 102 backend tests pass

### Frontend - Notification UI

- `NotificationBell` component with Radix UI Popover, red unread count badge, conditional polling fallback (`refetchInterval: isConnected ? false : 30_000`)
- `NotificationDropdown` with scrollable list, relative timestamps, unread indicator (blue dot), mark-as-read on click, mark-all-as-read button, empty state
- Metadata validated with `notificationMetadataSchema.safeParse()` (no unsafe type assertions)
- Click-to-navigate uses `router.push()` for client-side navigation

### Frontend - SSE Hooks

- Generic `useSSE` hook (`src/lib/use-sse.hook.ts`) — SSR-safe, native auto-reconnect, reusable for any SSE endpoint
- `useNotifications` hook wraps `useSSE` for `/notifications/sse`, invalidates tRPC queries on message
- `onMessage` callback wrapped in `useCallback` for referential stability

### Frontend - Layout

- Dashboard header bar now visible on all screen sizes (was mobile-only), contains `NotificationBell`
- Mobile hamburger menu and title hidden on desktop via `lg:hidden`

### Database

- `Notification` model with composite index `[userId, isRead]`
- `NotificationType` enum (EVENT_TRIGGERED, EVENT_RESOLVED, EVENT_SNOOZED, EVENT_REOPENED)
- Seed data includes 4 sample notifications (2 unread, 2 read) for admin user

- All types exported: `NotificationType`, `NotificationMetadata`, `Notification`, `MarkNotificationReadInput`, `NotificationListInput`, `UnreadCountOutput`
