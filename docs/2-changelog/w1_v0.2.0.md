# Changelog - Week 1, 01-04-2026, V. 0.2.0

**Release Date**: Week 1, 01-04-2026 at 15:42
**Version**: 0.2.0 (previously 0.1.0)
**Object**: feat: event trigger enhancements, snooze expiration cron, frontend pagination

## Changes

### Backend - Event Trigger Enhancements

- `EventsService.trigger()` now validates that the parent workflow is active before creating an event (throws `BAD_REQUEST` if inactive, `NOT_FOUND` if missing)
- Event creation wrapped in a `$transaction` that also creates an `EventHistory` entry with action `TRIGGERED`
- `NotificationsService.notify()` called after successful event creation with type `event.triggered`
- Router passes `ctx.user?.id` as `triggeredBy`; defaults to system user ID when absent

### Backend - Snooze Expiration Cron

- New `SnoozeExpirationService` runs every 5 minutes via `@nestjs/schedule` `@Cron()` decorator
- Queries for `SNOOZED` events with `snooze.until <= now()` and re-opens them in a transaction (status → `OPEN`, creates `REOPENED` history entry, deletes snooze record)
- Notifies via `NotificationsService` for each reopened event
- `@@index([until])` added to Snooze model for efficient cron queries

### Frontend - Event List & Pagination

- Pagination controls with Previous/Next buttons and "Page X of Y" indicator
- Page size set to 15 items; pagination bar sticks to the bottom of the viewport
- Status filter buttons reset page to 1 on change
- Mobile card layout for events on small screens

### Frontend - Event Detail (Mobile Responsive)

- Title and status badge stack vertically on mobile
- Comment form stacks vertically on small screens
- History entries stack vertically on mobile
- Payload section has horizontal scroll for overflow
- Added `min-w-0` to prevent flex children from expanding beyond viewport

### Frontend - Workflow Routing Fix

- Created `workflows/new/page.tsx` to prevent the `[id]` dynamic route from catching `/workflows/new`
- Removed `any` type from workflow list component, replaced with proper `Workflow` interface

### Database Seed

- Added 25 additional seed events (events 6-30) for pagination testing across 10 different servers and 3 workflows
