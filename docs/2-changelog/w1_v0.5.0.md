# Changelog - Week 1, 02-04-2026, V. 0.5.0

**Release Date**: Week 1, 02-04-2026 at 17:31
**Version**: 0.5.0 (previously 0.4.0)
**Object**: feat: end-to-end type safety improvements and error handling hardening

## Changes

### End-to-End Type Safety (Output Schemas)

- Added Zod output schemas for all 21 tRPC procedures across 4 routers (workflows, events, auth, notifications)
- Added `output` field to all `@Query()` and `@Mutation()` decorators for runtime output validation via nestjs-trpc
- Split `workflowOutputSchema` into strict base + `workflowWithCountSchema` (list) + `workflowDetailOutputSchema` (detail with events)
- Changed `z.unknown()` to `z.record(z.unknown())` for event payload fields in both event and workflow schemas
- Used `z.coerce.date()` for all date fields to handle Prisma Date objects and ISO string serialization

### Frontend Type Inference

- Created `apps/frontend/src/lib/trpc-types.ts` using `inferRouterOutputs<AppRouter>` for zero-duplication frontend types
- Deleted manual `workflow.types.ts` — all frontend types now derived from backend router definitions
- Updated `workflow-card.component.tsx` and `workflow-list.component.tsx` to use inferred types

### Error Handling Hardening

- Created `ErrorBoundary` React class component with default fallback UI and retry capability
- Added `Toaster` (sonner) for toast notifications on tRPC mutation errors via global `onError` handler
- Wrapped dashboard layout with `ErrorBoundary` + `Toaster`
- Changed `ResendEmailService` to throw `TRPCError` instead of generic `Error` for consistent error propagation

### Backend Improvements

- Removed unsafe `as unknown as Prisma.InputJsonValue` cast on workflow recipients
- Improved cookie typing in tRPC context with `RequestWithCookies` type alias
- Updated `@generated/index.ts` placeholder with output schemas for accurate frontend type inference

### Testing

- Added 20 output schema validation tests (`output-schemas.spec.ts`)
- Added 4 ResendEmailService error handling tests (`resend-email.service.spec.ts`)
- Added 4 ErrorBoundary component tests (`error-boundary.component.test.tsx`)
- Excluded test files from frontend `tsconfig.json` (no frontend test runner configured)
- All 127 backend tests passing
