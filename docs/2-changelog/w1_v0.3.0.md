# Changelog - Week 1, 02-04-2026, V. 0.3.0

**Release Date**: Week 1, 02-04-2026
**Version**: 0.3.0 (previously 0.2.0)
**Object**: feat: core domain model refinements, auth hardening, code review fixes

## Changes

### Backend - Auth & Security

- Increased JWT access token expiration from 15m to 1h (`JWT_EXPIRATION` default changed)
- Explicit `if (!ctx.user) throw UNAUTHORIZED` guards on `workflows.create` and `workflows.simulate` routers
- Removed unused `public.decorator.ts` and `jwt-cookie.guard.ts` files
- `ResendModule.forRootAsync` uses typed `ConfigService` factory with no hardcoded fallbacks
- `DailySummaryService` skips sending when `DAILY_SUMMARY_TO` is not configured (instead of failing)
- `DailySummaryService` uses `escapeHtml()` to sanitize workflow names in HTML email body
- `ResendEmailService.sendBatch` uses `Promise.allSettled` for resilient batch sending with null-safe `data?.id`

### Backend - Domain Model

- `EventHistory.action` converted from free `String` to Prisma enum (`EventAction`: CREATED, TRIGGERED, RESOLVED, SNOOZED, REOPENED)
- Matching `eventActionSchema` Zod enum exported from `@workflow-manager/shared`
- `deviationPercentage` max cap removed (was `.max(100)`, now unbounded for legitimate >100% deviations)
- Template variable placeholders standardized to `{{metric}}` and `{{value}}` across backend, frontend, and seed data
- `TriggerEvaluationService` template rendering uses `{{metric}}`/`{{value}}` variables

### Backend - Tests

- All 77 unit tests pass
- `DailySummaryService` tests updated: mock provides `DAILY_SUMMARY_TO`, new test for skip-when-unconfigured case
- `AuthService` spec updated for 1h token expiration

### Frontend - Auth (Next.js Middleware Token Refresh)

- `middleware.ts` enhanced with proactive server-side token refresh:
  - Decodes JWT `exp` claim (without cryptographic verification) to detect tokens expiring within 2 minutes
  - Calls backend `auth.refresh` server-side and forwards `Set-Cookie` headers transparently
  - Falls back to `/login` redirect if refresh fails
- `trpc.ts` simplified: removed all client-side refresh/retry logic (~50 lines), now a clean `httpBatchLink` with `credentials: 'include'`

### Frontend - UI Enhancements

- Delete/remove buttons replaced with trash icons across workflow form (recipient removal)
- Metric field uses native `<datalist>` for suggestions (cpu_usage, memory_usage, disk_usage, etc.) while allowing free text
- Snooze dialog uses two-step confirmation flow: select duration, then confirm with "Snooze for X" button
- Snooze and resolve mutations invalidate `events.findAll` so tables update immediately
- Back arrow navigation added to event detail page (consistent with workflow detail)
- `StatusBadge` component extracted for reuse across event list, detail, and workflow pages

### Frontend - Code Quality

- Double `triggerConfigSchema.safeParse` in workflow detail page consolidated to single variable
- `router.back()` replaced with `router.push('/events')` for consistent navigation
- `z.array(recipientSchema).catch([])` for safe recipient parsing
- `WorkflowForm` receives parsed `triggerConfig` and `recipients` props (no raw JSON)

### Database

- New `EventAction` enum type in PostgreSQL (migrated with raw SQL for existing data)
- Seed data uses `{{metric}}`/`{{value}}` template variables

### Shared Package

- `eventActionSchema` Zod enum exported from `@workflow-manager/shared`
- `recipientSchema` and `triggerConfigSchema` exported for frontend consumption
